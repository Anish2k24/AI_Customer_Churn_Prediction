# Contributing to AI Customer Churn Predictor

Thank you for your interest in contributing to the AI Customer Churn Predictor project! We welcome contributions from the community.

## How to Contribute

### 1. Fork the Repository
- Click the "Fork" button on the top right of this repository
- Clone your forked repository to your local machine

### 2. Create a Feature Branch
```bash
git checkout -b feature/YourFeatureName
```

### 3. Make Your Changes
- Ensure your code follows the project's style guidelines
- Add tests for new features
- Update documentation as needed

### 4. Test Your Changes
```bash
# Run the Streamlit app
streamlit run app.py

# Test model training
python train_model.py
```

### 5. Commit Your Changes
```bash
git add .
git commit -m "Add: Brief description of your changes"
```

### 6. Push to Your Fork
```bash
git push origin feature/YourFeatureName
```

### 7. Create a Pull Request
- Go to the original repository
- Click "New Pull Request"
- Select your feature branch
- Provide a clear description of your changes

## Development Guidelines

### Code Style
- Follow PEP 8 Python style guidelines
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions small and focused

### Testing
- Test your changes thoroughly
- Ensure the Streamlit app runs without errors
- Verify model training works correctly

### Documentation
- Update README.md if you add new features
- Add comments to complex code sections
- Update requirements.txt if you add new dependencies

## Types of Contributions

### Bug Fixes
- Fix bugs in existing code
- Improve error handling
- Fix typos and documentation errors

### New Features
- Add new ML algorithms
- Implement additional feature engineering techniques
- Enhance the dashboard with new visualizations

### Documentation
- Improve README.md
- Add code comments
- Create tutorials or examples

### Testing
- Add unit tests
- Improve test coverage
- Test with different datasets

## Code of Conduct

Please be respectful and constructive in all interactions. We aim to create a welcoming environment for all contributors.

## Questions?

If you have questions about contributing, feel free to open an issue or contact the maintainers.
